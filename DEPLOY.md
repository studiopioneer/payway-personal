# Деплой PayWay Personal — инструкция для Cowork

## ВАЖНО: Как деплоить
Деплой происходит АВТОМАТИЧЕСКИ через GitHub Actions при пуше в ветку `main`.
НЕ нужно деплоить через терминал, FTP, SSH или любые другие ручные способы.
НЕ ИСПОЛЬЗУЙ терминал для git команд.

## Порядок деплоя

### Шаг 1: Внеси изменения в файлы
Редактируй PHP, Vue-компоненты, стили — что нужно.

### Шаг 2: Если менял файлы в `src/` — собери фронтенд
```bash
npm run build
```
Это обновит `assets/`. Включи обновлённую папку `assets/` в коммит.

### Шаг 3: Запуши изменения через GitHub MCP
Используй инструмент `push_files` из GitHub MCP-коннектора:
- owner: `studiopioneer`
- repo: `payway-personal`
- branch: `main`
- message: описание изменений
- files: массив изменённых файлов с их содержимым

ВСЕГДА используй GitHub MCP инструмент `push_files` для отправки файлов.
НИКОГДА не используй терминал для `git add`, `git commit`, `git push`.
Это критически важно — терминальный git медленный и ненадёжный в Cowork.

### Шаг 4: Дождись автодеплоя
После пуша GitHub Actions автоматически загрузит файлы на тестовый сервер через FTP.
Время деплоя: ~2 минуты. Ничего дополнительного делать не нужно.

## Структура проекта
- `payway-personal.php` — главный файл плагина
- `functions.php` — функции
- `activation.php` — активация плагина
- `admin/` — админ-панель
- `includes/` — PHP-классы и хелперы
- `pages/` — страницы
- `assets/` — собранный Vue-бандл (результат `npm run build`)
- `src/` — исходники Vue (НЕ деплоятся на сервер)
- `style.css` — стили
